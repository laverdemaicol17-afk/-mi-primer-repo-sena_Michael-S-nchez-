  # Notas de la práctica
   - Aprendí a crear carpetas en Visual Studio Code.
   - Aprendí a crear archivos dentro de un repositorio.

     ¿Qué fue lo que más te gustó de trabajar con GitHub Desktop?
     rta; Lo que más me gustó fue que GitHub Desktop facilita mucho el uso de Git sin necesidad de usar comandos en la terminal. Su interfaz es muy intuitiva: puedes ver los cambios, hacer commits, crear ramas y subir proyectos con solo unos clics. Además, ayuda a visualizar el historial y los archivos modificados, lo que hace el trabajo en equipo más ordenado y claro.
   - ¿Qué te pareció más difícil?
   rta; Lo más difícil al principio fue entender el flujo de trabajo entre local y remoto, es decir, cuándo usar fetch, pull, commit y push. También puede ser confuso resolver conflictos de fusión (merge conflicts) cuando varias personas modifican el mismo archivo.
   - ¿Para qué crees que sirve un archivo .gitignore?
   rta; El archivo .gitignore sirve para indicar qué archivos o carpetas no deben subirse al repositorio. Por ejemplo, puedes excluir archivos temporales, configuraciones locales, contraseñas o dependencias que no son necesarias para los demás. Así el repositorio se mantiene limpio y seguro.
   - ¿Por qué es importante escribir buenos mensajes en los commits?
   rta; Es importante porque los mensajes de los commits explican los cambios realizados. Si se escriben bien (claros, breves y descriptivos), facilitan entender la evolución del proyecto, detectar errores y colaborar con otros. En equipos grandes, buenos mensajes de commit ahorran mucho tiempo al revisar el código o buscar la causa de un problema.
